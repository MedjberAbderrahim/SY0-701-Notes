# Benefits of Automation
- Saving time, i.e. do not type manually, use PC's processing power in task, ...
- Automatic baseline Enforcement
- Standardization of infrastructure configuration, e.g. router configs, ...
- Secure/Seamless scaling
- Simplify tasks for employees & direct them toward more interesting stuff
- Improve reaction time (PCs are faster than human)
- Always on, 24/7 active time, a good workforce multiplier
# Cases of Automation
- User provisioning (on-boarding, off-boarding, permission management, ...).
- Guard rails (automatic user fault detection, e.g. GitHub automatically warns against pushed API-like keys).
- Security groups, automatic assignment/removal/removal from certain groups.
- Ticket creation, for help-desk people.
- Escalation (automatically correcting emerging problems before they get worse)
- Controlling services & access (automatic management of services)
- Continuous integration & testing (automatic push, testing, deployment)
- Integrations and **APIs (Application programming interfaces)**: manual use of APIs and interaction with third-party devices/services
# Automation Considerations
- Complexity: requires granular development and extensive testing
- Cost: to develop & implement
- SPOF (Single Point of Failure): poses a SPOF issue when it stops
- Technical Debt: creating scripts to 'cover' problems increases technical debt
> Technical Debt refers to **the future costs associated with relying on shortcuts or sub-optimal decisions made during software development**
- Ongoing Supportability: maintenance costs
