## Firewall
- Filter traffic by port or application
- Can act as VPN concentrator for site-to-site VPNs, and as routers
### Ports/Protocols
- Filter traffic by port (traditional) or application (NGFW)
- For traditional, it can filter:
	- tcp/80, tcp/443
	- tcp/22
	- ...
- For NGFW, it can filter:
	- HTTP, HTTPs
	- SSH
	- ...
### Rules
- Logical path, from up (more specific/accurate/exact rules) to bottom (more generic/broad rules)
- Generally ends with a Deny-All at the bottom (least prioritized), to prevent any traffic that wasn't accounted for
- The rule base or policy list is generally called an **ACL (Access Control List)**

### Screened Subnet (Demilitarized Zone (DMZ))
- Screened subnet holds services that should be available & accessible from the internet
- Should be monitored in a way to not give access to internal networks, only to them (the DMZ)
## IDS/IPS
- Usually/Can be integrated into an NGFW
- Detect malicious traffic based on signatures, or anomalies (out of normal traffic from the pre-built "normal" baseline)
### Rules
- Can specify actions (block, allow, alert, ...)
- Generally contains thousands of grouped/individual rules, must find correct balance between security & false-positives
## Web Filter
### Content Filtering
- Control traffic based on data within the content (URL, website category, ...)
- Filter against inappropriate content (not safe for work, parental control, ...)
### URL (Universal Resource Locator) scanning
- Allow/Refuse URLs in lists
- Manages URLs in categories for ease-of-use (Travel, Malware, ...)
- Can have limited control capabilities, they are generally integrated into NGFWs
### Agent Based
- Always-On, agent-based content filters managed by a central management platform
- Must all be regularly updated
### Proxies
- Sits between the users & external networks, useful for caching, access control, URL filtering, ...
- Some proxies must be explicitly exposed to some applications (explicit), whereas others integrate seemingly (transparent)
### Block Rules
- Different actions on sites based on their URLs or categorization
### Reputation
- Decide action (allow, block, ...) of URLs based on perceived risk (reputation)
- Websites are scanned & assigned a reputation (manual/automatic)
## DNS Filtering
- Fetches IP address of domain name before connecting
- Updated with real-time threat intelligence, doesn't resolve harmful domain names
## Operating system security
### Active Directory
- A primarily windows-based database of everything (computers, user accounts, ...) on the network
- Enforces centralized authentication, access control, ... 
### Group Policy
Microsoft Windows feature and Active Directory (AD) tool used to enforce security configurations, settings, and restrictions across computers and users in a network. It is critical for maintaining security baselines, such as password policies, account lockout, and firewall settings, to ensure consistent security posture.
### SELinux (Security-Enhanced Linux)
- Open source packages that applies security patches to the Linux kernel, implements **[MAC (Mandatory Access Control)](https://en.wikipedia.org/wiki/Mandatory_access_control)**, instead of the traditionally used **[DAC (Discretionary Access Control)](https://en.wikipedia.org/wiki/Discretionary_access_control)**
- Limits application access by implementing Least Privilege
## Implementation of Secure Protocols
### Protocol Selection
- Always use secure protocols which implement encryption
- Else you'll end up in the ["wall of sheep"](https://www.phvillage.io/pages/wall-of-sheep/) at [DEFCON](https://defcon.org/#) lol
![[Pasted image 20260204210359.png]]
### Port Selection
- Some ports are known to be default ports for unsecure protocols, e.g. 80 for HTTP
- Not a guaranteed way to evaluate security, e.g. can use HTTP on port 443 (non standard)
### Transport Method
- Good for application-independent encryption, e.g. WPA2/3, VPN, ...
## Email Security
- Email protocols aren't so secure by default, easy to spoof messages
### Mail Gateway
- The gateway for the mail of the organization
- If it's on premises, must be put on a screened subnet (DMZ)
### SPF (Sender Policy Framework)
- Configures a list of servers authorized to send emails on behalf of the domain
- Adds that list to a DNS TXT record
![[Pasted image 20260204212438.png]]
In here, if a mail gateway receives a mail under the name `professormesser.com`, from `mailgun.org`, it is trusted
### DKIM (DomainKeys Identified Mail)
- Mail server signs all outgoing mail (digital signature), and mentions the public key in the DKIM TXT record (DNS)
![[Pasted image 20260204212814.png]]
### DMARC (Domain-based Message Authentication Reporting and Conformance)
- An extension of **SPF (Sender Policy Framework)** & **DKIM (DomainKeys Identified Mail)**, specifies what actions to do when the mail server meet a mail that fails the SPF & DKIM verifications
- Can be configured through the host DNS (of the original mail owner) server to allow, spam, reject the mail, and a destination for compliance reports, which allows for statistics & monitoring

## FIM (File Integrity Monitoring)
- Always monitor files that shouldn't change (OS/Applications files, ...)
- Can be done with **SFC (System File Checker)** on windows or **Tripwire** on linux
- Can also be implemented in a HIPS (Host-based IPS), IPS, IDS, ...
## DLP (Data Loss Prevention)
- Mentioned [here](4.%20Security%20Alerting%20and%20Monitoring%20Concepts%20and%20Tools.md#DLP%20(Data%20Loss%20Prevention)).
- Can be in the computer (endpoint DLP) for **data in use**, on the network for **data in transit**, or on the server for **data at rest**.
- Can even be used for USBs or emails and be cloud-based.
## Endpoint
- Consists of the device of the user, e.g. laptop, desktop, mobile, ...
- Prime target for attack, must be monitored for inbound & outbound traffic, so we must implement a multi-faceted security, i.e. **[Defense in Depth](https://en.wikipedia.org/wiki/Defense_in_depth_(computing))**
## Edge vs Access Control
Edge control is controlling at the point where the external network meets the internal network, usually protected through a firewall.
While Access Control is at the device-level, and can be more granular, e.g. by user, location, application, ...
## EDR (Endpoint Detection & Response)
- Implements advanced analysis & detection methods (behavioral analysis, machine learning, ...) instead of signatures for scalable protection against ever-evolving threats using lightweight agents on the endpoint.
- Features **[root-cause analysis](https://www.splunk.com/en_us/blog/learn/root-cause-analysis.html)** which also allows for identifying reason of infection with malware, and how to prevent it from spreading to other systems.
- Fully automated response, i.e. isolation, quarantine, rollback, ...
## XDR (Extended Detection & Response)
- EDR on steroids
- Evolution of the **EDR (Endpoint Detection & Response)**, improve missed detections, false positives, ...
- Correlates between multiple sources (endpoints, networks, ...) for multi-system, network-based protection
## User Behavior Analytics
- Used by **XDRs (Extended Detection & Response)** to better detect threats
- Builds a baseline for **usual** user behaviors, network hosts
- The built baseline allows for detection of **"unusual"** activities using multiple methods, e.g. set of rules, pattern matching, statistical analysis, ...
