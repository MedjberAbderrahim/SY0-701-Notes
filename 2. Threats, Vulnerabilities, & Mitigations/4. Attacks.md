## Malware
-  **Malware**: Broad term for malicious code that steals data, encrypts files, or creates **botnets**.
-  **Propagation**: Occurs through user action (clicks), **drive-by downloads** (automatic), or exploiting known vulnerabilities.
-  **Defense**: Prioritize **patching**, updated antivirus signatures, and **offline backups** to prevent ransomware from reaching backup data.
### **Types**
-  **Viruses**: Require **human intervention** to spread (e.g. clicking a link).
    -  **Fileless Virus**: Operates in **system memory** and the registry to avoid disk-based detection.
-  **Worms**: Self-replicating; spread across networks **without user action**.
-  **Ransomware**: Encrypts data for payment (usually cryptocurrency).
-  **Spyware**: Monitors browsing and activity for affiliate fraud or data theft.
-  **Keyloggers**: Capture keystrokes, screenshots, and clipboards, bypassing network encryption by attacking the source.
-  **Logic Bombs**: Triggered by a **specific event** (date/time or user login). They are hard to detect because they lack standard signatures.
-  **Rootkits**: Hide within the **OS kernel** to remain invisible. **UEFI Secure Boot** prevents them by verifying the kernel signature before booting.
-  **Bloatware**: Unnecessary manufacturer-installed software that may introduce vulnerabilities on new devices.
### **Mitigations**

| Threat          | Key Defence Strategy                                       |
| :-------------- | :--------------------------------------------------------- |
| **Ransomware**  | Maintain **offline** backups.                              |
| **Worms**       | Use firewalls and **Intrusion Prevention Systems (IPS)**.  |
| **Logic Bombs** | Monitor core file changes and enforce **least privilege**. |
| **Rootkits**    | Enable **Secure Boot** in UEFI BIOS.                       |
| **Spyware**     | Use specialised tools like Malwarebytes.                   |
## Physical Attacks
-  Physical Access to hardware generally implies access to the device
-  RFID Cloning: Cloning access badge, key fobs, ...
-  Brute Force: literally breaking doors, locks, ...
-  Environmental Attack: if you couldn't attack the target, attack the environment supporting it, e.g.: Electricity, HVAC (Heating, Ventilation, and Air Conditioning), ...
## **Network Attacks**

-  **DNS Poisoning**: Attackers modify **DNS servers**, local host files, or domain registrations to redirect users to malicious IP addresses.
-  **URL Hijacking (Typosquatting)**: Registering domains that are misspellings of legitimate sites (e.g., _messir.com_) to steal credentials or generate ad revenue.
-  **DoS and DDoS**: **Denial of Service (DoS)** overloads a system to force failure. **Distributed DoS (DDoS)** uses global **botnets** to overwhelm targets asymmetrically.
-  **Amplification Attacks**: Attackers send small queries (DNS, NTP, ICMP) with a **spoofed victim IP**, resulting in massive response data hitting the victim.
-  **On-path (Man-in-the-Middle)**: An attacker sits between two devices to monitor or modify traffic in real-time. **ARP Poisoning** accomplishes this on local subnets by spoofing MAC addresses.
-  **Wireless Attacks**:
    -  **Deauthentication**: Sending unencrypted management frames to disconnect users from an access point.
    -  **RF Jamming**: Creating intentional radio interference to decrease the **signal-to-noise ratio**, preventing communication.
## **Application Attacks**

-  **Injection**: Adding malicious code (SQL, HTML, XML, LDAP) into application input fields that lack proper validation.
-  **Buffer Overflow**: Writing more data than a memory buffer can hold, causing it to spill into adjacent memory to crash apps or execute code.
-  **Privilege Escalation**: Exploiting vulnerabilities to gain **Vertical** access (Administrative) or **Horizontal** access (another user's data). Mitigated by **ASLR** (memory randomization) and **DEP** (Data Execution Prevention).
-  **CSRF (Cross-Site Request Forgery)**: Tricking a user's trusted browser into sending unauthorized requests to a website they are currently logged into.
-  **XSS (Cross-Site Scripting)**: Injecting malicious scripts into web pages to steal sensitive information like credit card data.
-  **Directory Traversal**: Using `../` sequences to access files outside the intended web server directory.
## **Cryptographic Attacks**

-  **Birthday Attack**: Finding two different plaintexts that produce the same hash, known as a **hash collision**. This vulnerability made **MD5** obsolete after researchers created fraudulent certificates.
-  **Downgrade Attack**: Forcing a system to use a weaker encryption protocol or none at all.
-  **SSL Stripping**: A specific downgrade attack where an on-path attacker intercepts a request and forces the victim to use **HTTP** instead of **HTTPS**.
## **Password Attacks**

-  **Hashing vs. Plaintext**: Passwords should always be stored as **hashes** (fixed-length fingerprints) rather than plaintext to prevent reversible theft.
-  **Brute Force**: Attempting every possible character combination until the correct hash is found. **Offline attacks** are more common as they avoid account lockouts.
-  **Password Spraying**: Attempting a few **common passwords** (e.g., "123456") against many different accounts to bypass lockout thresholds.
-  **Replay Attacks**: Capturing authentication data (like a password hash) and re-sending it to a server to pose as the victim.
-  **Session Hijacking (Sidejacking)**: Stealing a victim's **Session ID** from browser cookies to gain account access without needing credentials.
## **IOCs (Indicators of Compromise)**

An **Indicator of Compromise (IOC)** is evidence providing high confidence that a system has been breached. This evidence allows security professionals to track attacker progress and identify active threats.
### **Authentication and Account Indicators**

- **Account Lockouts**: Unusual lockouts from failed password attempts or accounts being **administratively disabled** without a corresponding task.
- **Impossible Logins**: Logins occurring from geographically distant locations (e.g. Nebraska and Australia) within a timeframe that is physically impossible to travel.
- **Concurrent Logins**: One person logged in from multiple facilities or locations simultaneously, though this may sometimes be legitimate multi-device usage.
### **Network and Resource Indicators**

- **Resource Consumption**: Sudden, unusual spikes in network traffic (e.g. at 3:00 am), which often indicate **data exfiltration** or large file transfers.
- **Inaccessible Resources**: Servers crashing during an exploitation attempt, or network disruptions caused by massive data transfers.
- **Traffic Anomalies**: Spikes in international traffic for a domestic-focused organization or modified **DNS information**.
### **System and Software Indicators**

- **Integrity Changes**: Changes in **file hash values** (indicating modified files) or sensitive files being read more frequently than normal.
- **Disabled Security**: Malware frequently disables **antivirus updates** or security patches to prevent the closing of vulnerabilities and maintain persistence.
- **Data Leakage**: Sensitive organizational data suddenly appearing on the public internet, often used as leverage in **ransomware** "double extortion" schemes.
### **Log Irregularities**

- **Out-of-Cycle Logging**: Security patches, applications, or traffic flows appearing outside of the standard **change control schedule**.
- **Deleted Logs**: Attackers often clear logs to hide their authentication and file transfer history; missing log data is a major warning sign.
