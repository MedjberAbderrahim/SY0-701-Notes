## Infrastructure Considerations
### Device Placement
-  Even though different companies/fields have fundamentally different network designs, some principles and patterns stay the same.
-  Firewalls must be set between zones, to secure access from & to those zones.
-  Other devices must also be used, e.g. honeypots, jump servers, load balancers, sensors, ...
### Security Zone
-  Different than IP Subnet, Logical segmentation which separates devices based on their use or access-type
-  Each area of the network can be split depending on its properties, e.g. Trusted / Untrusted, Internal / External, ...
-  Helps simplify security rules, e.g.
	-  **ALLOW(Trusted -> Untrusted)**
	-  **ALLOW(Unrusted -> Screened)**

### Attack Surface
-  Defined as the openings in a network that can be attacked and gained access through by hackers.
-  Everything can be/have a vulnerability, e.g. Application Code, Open Ports, Human Error, ...
-  Always minimize the attack surface; audit the code, block ports in the firewall, ...
### Connectivity
-  Connectivity is part of the attack source, as all all devices connected may be entry points to each other through lateral movement.
-  We can secure the network cabling & ports, physically & logically.
-  Use Encryption, Application Level (HTTPS, ...), or Network Level (IPSec tunnels, VPNs, ...)
### Failure Modes
-  Devices/Services eventually fail, there are 2 modes of failure possible:
	-  **Fail-Open:** Network remains up & running, albeit the lack of security processes.
	-  **Fail-Closed:** Network connection severed, and no communication on that link.
-  Security Devices are **generally fail-Closed**, but checking out the documentation is necessary.
### Device Attribute
#### Active vs Passive
There are 2 modes when implementing an IPS:
##### Active Monitoring
-  Blocks bad packets in real time.
-  Some organizations don't implement it in fear of outage causing downtime (fail-closed), or IPS being too aggressive.
##### Passive Monitoring
-  Referred to as IDS Design, even though it's in IPS, because it's not able to block traffic in real time.
-  Devices communicate freely, IPS can't prevent intrusions.
#### Inline vs. Tap/Monitor
##### Inline
-  Placed Inline between external network & internal network.
-  Some organizations don't implement it in fear of outage causing downtime (fail-closed).
##### Tap/Monitor
-  A copy of traffic is sent to the IPS by the switch, through **a Port Mirror (SPAN - Switch Port ANalyzer)**, or a physical network tap.
#### "Active vs Passive" vs "Inline vs. Tap/Monitor"
-  **"Active vs Passive":** about **what the security tool does to traffic**.
> i.e. Does it interfere with traffic?
-  **Inline vs Tap/Monitor:** about **how traffic reaches the device**.
> i.e. Does traffic flow through it?
### Network Appliances
#### Jump Server
-  A **hardened** device **inside the network**, that is **accessible from the outside**.
-  The External client would connect to the jump server, and then from it to the internal servers.
-  A Compromised jump server is very dangerous, as it could give access to internal resources.
#### Proxy Server
-  Sits between users & external networks **as a middle-man**, inspecting traffic and ensuring it's not malicious
-  Generally divided into **2 types by visibility**:
	-  **Explicit Proxy:** needs to be configured in the application and/or OS.
	-  **Invisible Proxy (transparent):** Nothing configured on the host machine, hence it's invisible to the user.
-  Generally divided into **2 types by level**:
	-  **Network Proxy:** the simplest forms of "proxy" is NAT (i.e. middle man, not actual proxy)
	-  **Application Proxy**: Most proxies are application proxies, which means they understand Application data (HTTP, HTTPs, ...)
-  Generally divided into **2 types by functionality**:
	-  **Forward Proxy (internal proxy)**: Protects & Controls user access to the internet.
	-  **Reverse Proxy:** Protects inbound traffic to a server/service, can cache content.

-  **Open Proxy:** Proxy servers that are usable by anyone on the internet (public), they pose a significant security concern, as they're generally used to circumvent security controls, but is unsecure bcz we don't know who controls that proxy and we can't verify its security.
#### Load Balancer
-  Distributes the load between multiple servers, while remaining invisible to the end user, generally used in large scale implementations.
-  Server outages are easily fixable thanks to the fast convergence, which makes the load balancer fault tolerant.
-  Other than distributing traffic, certain load balancers can even perform:
	-  **TCP Offloading:** doesn't re-do TCP Handshake with the server each time.
	-  **SSL Offloading:** Strips down the SSL encryption for the server.
	-  **Caching:** to ensure fast response.
	-  **Prioritization:** QoS, ...
-  Load balancers generally work in 2 modes:
	-  **Active/Active:** All others are active at same time, allowing for high availability.
	-  **Active/Passive:** Some servers are up & used, while others & down & **usable**, when one of an active server falls, one of the passive devices replaces it.
#### Sensors & Collectors
-  Sensors generate logs & information for statistical & security analysis, they are implemented in a lot of places, e.g. routers, switches, Firewalls, ...
-  Sensors' logs are to be collected by collectors, e.g. Proprietary consoles (IPS, Firewall), SIEM Consoles, or syslog servers
-  Which then is compared, analyzed, and treated accordingly
### Port Security
The most common authentication method used is username+password, commonly used in wireless & wired networks.
#### EAP (Extensible Authentication Protocol)
-  A widely used authentication protocol, which adds many different authentication methods to devices.
-  EAP Generally integrates with IEEE 802.1X, which in result, prevents access to network until the authentication succeeds.
#### IEEE 802.1X
-  Port-based Network Access Control (**PNAC** or **NAC**), uses EAP to prevent access to network until authentication occurs.
-  Used in conjunction with authentication databases, e.g. RADIUS, LDAP, Kerberos...
-  The process mainly involves 3 components:
	-  Supplicant: The client wanting to connect
	-  Authenticator: The device that provides access (switch, ...)
	-  Authentication server: the server that stores credentials, e.g. RADIUS, ...
#### Authentication Process
1.  When a **supplicant** (the client) initially connects to the network, the **authenticator** (switch or wireless access point) blocks all access until the authentication process is completed.
2.  The authenticator initiates the exchange by sending an **EAP request** to the supplicant, asking for login credentials.
3.  The supplicant provides an **EAP response** containing its identity (device name), which the authenticator then passes to the **authentication server**.
4.  If the authentication server is accepting logins, it sends a request back through the authenticator asking for specific **additional details** or credentials.
5.  The authenticator forwards this request to the supplicant, which then provides the required **login credentials** (e.g. username and password).
6.  These credentials are sent by the authenticator to the authentication server for final verification.
7.  If the credentials match the database (e.g. **RADIUS**, **LDAP**, or **Active Directory**), the server replies with a **successful login** message.
8.  The authentication server then instructs the authenticator to **allow access** to the network, completing the port-based security process.
### Firewall Types
-  Firewalls are used in our home, office, and even OS
-  Inspects all outbound & inbound traffic, blocking inappropriate content, and protecting against evil.
-  Can be classified by its layer (layer 4 vs layer 7), and modernity (traditional vs NGFW)
-  Can encrypt traffic using VPNs, and work as routers, in terms of placement, NAT functionality, and authentication of dynamic routing communications.
#### WAF (Web Application Firewall)
-  Web Application Focused firewall, allows/denies based on expected input.
-  Specifically designed to stop web application attacks (SQL Injections, XSS, ...)
-  A major focus of **PCI DSS (Payment Card Industry Data Security Standard)**.
#### UTM (Unified Threat Management) / Web Security Gateway
An "all-in-one" security appliance that consolidates multiple security and networking functions into a single box, some of its features are:
-  URL Filtering / Content Inspection
-  Malware Inspection
-  Spam Filter
-  [CSU/DSU](https://en.wikipedia.org/wiki/CSU/DSU): allows the UTM appliance to plug directly into the digital leased line. This eliminates the need for an external, standalone CSU/DSU unit between the carrier's line (at the demarcation point) and the customer's router/UTM.
-  ...
#### NGFW (Next Generation FireWall)
-  One of the most modern types of firewalls, operates at OSI Layer 7 (Application Layer), which allows it to inspect packets/data very thoroughly (up to the application level).
-  Goes by other names, e.g. Application Layer Gateway, Stateful Multilayer Inspection, Deep Packet Inspection.
-  Controls traffic based on the application (Microsoft SQL Server, Twitter, ...), applies URL Filters, Website traffic control by category (e.g. gambling sites, ...).
-  Identifies applications & apply application-specific vulnerability signatures to the traffic.
## Secure Communication/Access
### VPN (Virtual Private Network)
-  Creates a tunnel to send data securely in an unsecure network (the internet).
- Encryption/Decryption happens through a VPN Concentrator, which is often integrated into a firewall, specialized cryptographic hardware, or even user applications
-  The concentrator can be a software solution usable/connectable via client software.

### Tunneling
- VPN Securely sends data by establishing a tunnel, where it encrypts the data and changes headers.
![[Pasted image 20260113231703.png]]
#### SSL/TLS (Secure Sockets Layer / Transport Layer Security) VPN (Remote Access VPN)
-  Uses SSL/TLS (tcp/443), reduces firewall issues, also it doesn't require Digital Certificates or shared passwords like IPsec
-  Usually used for remote access from a single client, can even have a light version run inside browser.
-  Generally used by devices wanting to communicate with a VPN Concentrator from a public network
![[Pasted image 20260113232559.png]]
#### IPsec VPN (Site-to-Site VPN)
Used by multi-site companies to connect their remote sites together using a secure IPsec Tunnel, with their firewalls **generally** acting as the VPN Concentrators on each side.
![[Pasted image 20260113233000.png]]
#### SSL/TLS VPN vs IPsec VPN
-  **Connection:** User-to-network (Remote) vs. Network-to-network (Site-to-Site).
-  **Software:** Client on each device (Remote) vs. Gateway-based (Site-to-Site).
-  **Access:** On-demand (Remote) vs. Always-on (Site-to-Site).
-  **Use Case:** Remote workers (Remote) vs. Connecting branch offices (Site-to-Site).
### SD-WAN (Software Defined Networking in a Wide Area Network)
Came to adapt to the dynamicity & flexibility the cloud has brought, which made traditional methodologies slow and inefficient with the cloud. 
-  **BEFORE (Traditional, Centralized Architecture):**
the traditional design of data centers set the data center as the central point of communication, it is in the same or close to the network of the databases & the service servers.
![[Pasted image 20260113234415.png]]
-  **AFTER (Modern, Decentralized Cloud-Based Architecture):**
The decentralization the cloud introduced makes this design inefficient, as it takes more hops to reach the required resources.
![[Pasted image 20260113234519.png]]

Remote sites can communicate directly with the resource they need through the appropriate network connection over the SD-WAN.
![[Pasted image 20260113235046.png]]
### SASE (Secure Access Service Edge)
-  **Problem:**
Now that we have decentralized services, each in different location in the cloud, how to secure them all and apply our VPN connection to them all ?
-  **Solution:**
	-  SASE (kinda like a "Next generation" VPN), which is **deploying security solutions in the cloud** to **protect cloud-based Services**, which will be close to them.
	-  Installing SASE Clients on local devices to be able to communicate to the cloud **securely**.

## Selection of Effective Controls
-  Many different security options make it hard to make the right choice, an organization might use:
	-  SSL/TLS VPN for remote access, while IPsec Tunnels for Site-to-Site access
	-  SD-WAN to provide seamless connection to cloud-based services.
	-  SASE to secure communication over the SD-WAN.
-  Each technology has its Pros & Cons, their usage depends on implementation of the application and the connectivity that's being used.
