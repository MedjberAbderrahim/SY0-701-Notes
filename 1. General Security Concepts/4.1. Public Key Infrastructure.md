**PKI** refers to the:
-  Policies
-  Procedures
-  Hardware
-  Software
-  People
that are related to managing digital certificates
## Symmetric Encryption
-  Same **shared key** for both encryption and decryption.
-  Can be challenging to safely share key across communicators.
-  Very fast to use thanks to low overhead
-  Often combined with asymmetric encryption.
## Asymmetric Encryption
-  2 **distinct, mathematically-related** keys, one is private while the other is public.
-  only the private key can decrypt data encrypted using the public key
### The Key Pair
-  Only used in **Asymmetric Encryption**.
-  Both built at the same time, lots of randomization, math, and large prime numbers.
-  Data is encrypted using the public key of the destination, while decrypted using the pair of that public key (the private key).
## Key Escrow
-  The key pairs are held by a 3rd party, outside/inside the organization.
-  Seems controversial but widely used, especially upper management in companies and governments.
