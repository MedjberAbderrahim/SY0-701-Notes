## Virtualization
 Cloud Providers Host/manage a lot VMs at the same time, if one could jump from 1 VM to another VM, big amounts of data could potentially be leaked, some risks:
-  OS Escape
-  Memory re-use: in case of dynamic memory, changing the allocated **shared** memory between the VMs can cause vulnerabilities, where 1 VM accesses the data of another VM.
## Cloud
Most companies nowadays use cloud for their apps, infra, ...; which places huge value on cloud security, some risks:
-  DOS/DDOS
-  Authentication Misconfiguration
-  Directory Traversal
-  Unpatched systems
-  Application vulnerabilities: allows access to system, which may affect the cloud instance.
## Supply Chain
Vulnerabilities can be at any point of the chain of software/service:
-  Service providers: [Target Corp. Breach - November 2013](https://redriver.com/security/target-data-breach).
-  Hardware providers: [DHS Arrests Reseller CEO - July 2022](https://www.justice.gov/usao-nj/pr/ceo-dozens-companies-and-entities-charged-scheme-traffic-estimated-1-billion-fraudulent).
-  Software Providers: [SolarWind Orion Supply Chain Attack](https://www.fortinet.com/resources/cyberglossary/solarwinds-cyber-attack).
## Misconfiguration
-  Open Permissions: [Verizon Open Amazon S3 Bucket](https://zvelo.com/lessons-learned-verizon-data-breach/).
-  Unsecured Admin Accounts.
-  Insecure Protocols: Telnet, FTP, SMTP, IMAP, HTTP, ...
-  Default Settings: [Mirai Botnet](https://www.cloudflare.com/learning/ddos/glossary/mirai-botnet/).
-  Open Ports & Services.
## Mobile Devices
-  Jailbreaking/Rooting: Rooting in Android, Jailbreaking in Apple IOS.
-  Sideloading: Installing software from outside the specified Sources/App Markets.
