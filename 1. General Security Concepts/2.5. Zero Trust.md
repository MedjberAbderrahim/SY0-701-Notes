Many networks traditionally have a relatively open internal environment once past the firewall.
## Concept
-  Covers every device, every process, every person.
-  Assume Nothing is inherently trusted.
-  Uses multi-factor authentication, encryption, system permissions, additional firewalls, monitoring and analytics, ...etc.
## Planes of Operation
Split the network into functional planes, applies to physical, virtual, and cloud components (basically anything network).
### Data Plane
Is the part of the device that's performing the **actual** security process, can be a switch, router, or a firewall.
-  Process the frames, packets, and network data.
-  Processing, forwarding, trunking, encrypting, NAT.
### Control Plane
Where we manage all the actions occurring in the data plane.
-  Define policies and rules.
-  Determines how packets should be forwarded.
-  Routing tables, session tables, NAT tables.
### Notes
Both physical switches and virtual devices (like virtual firewalls) can be segmented into data and control planes, ensuring flexible deployment across physical, virtual, and cloud environments. 
## Zero Trust Mechanisms
### Adaptive Identity
-  Consider the source and the requested resources.
-  Multiple Risk Indicators (Relationship to the organization, Physical Location, Type of Connection, IP Address, ...etc)
### Threat Scope Reduction
-  Decrease the number of possible entry points as much as operationally possible.
### Policy-driven Access Control
-  Combine the Adaptive identity with a predefined set of rules.
### Security Zones
Organize network segments into security zones based on trust levels:
- Untrusted zones (public internet)
- External zones
- Internal trusted zones (corporate offices, data centers)
Interests in **where you are coming from**, and **where you are going**.
### Policy Enforcement Components
1. **Policy Enforcement Point (PEP):** Inspects traffic and enforces policies.
2. **Policy Decision Point (PDP):** Evaluates requests against policies, consists of:
	1. **Policy Engine:** Decides whether to grant access.
	2. **Policy Administrator:** Implements decisions by communicating with the **PEP**, providing credentials or tokens.
### Zero Trust Workflow
1. Request originates from an untrusted zone.
2. Traffic passes through PEP.
3. PEP sends info to PDP.
4. Policy Engine evaluates request.
5. Decision sent to Policy Administrator.
6. If approved, PEP grants access.
This process ensures ongoing verification and strict access control.
