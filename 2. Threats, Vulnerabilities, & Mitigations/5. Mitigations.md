# Segmentation & Access Controle
## Segmentation
-  Segmenting the network physically, logically (VLANs, IP Subnets, ...), or virtually (like in the cloud)
-  Can improve performance, security, and ensure compliance with specific policies
## Access Control
-  Access Control Lists (ACLs), a simplified example of a firewall, can (dis)allow traffic based on customizable Criterias (Source/Destination IP/MAC/Port, Roles, ...)
-  Be careful of locking yourself out or locking critical services.
## Application Allow/Deny List
-  Allows/Denies certain software from running by their specific properties, often built-in to the OS management.
-  Those properties include Hash, Certificates, Path, ...
# Mitigation Techniques
## Patching
-  Can be monthly, or irregular, third-party updates; They ensure system stability & security.
-  Generally a manual process, as it's best to test the updates before mass-rolling them into the company.
## Encryption
-  Encrypting files/disks is important for data confidentiality.
-  Types are:
	-  File Level Encryption: Windows EFS.
	-  Full Disk Encryption (FDE): Encrypts everything on Disk, even the OS itself, BitLocker (Windows), FileVault (MacOS), ...
	-  Application Encryption: Encryption managed by the application itself.
## Monitoring
-  Generate logs using sensors on devices (IDS, IPS, Firewall logs, **real hardware sensors**, ...)
-  Collect & Manage them using SIEMs.
## Least Privilege
-  Always run applications in the minimum permissions possible, to avoid problems if they are compromised.
-  Minimize Attack surface (open ports, applications, services, ...).
## Configuration Enforcement
-  Always ensure everything is connected and working as expected, all apps are up-to-date.
-  Systems out of compliance are quarantined (Private VLAN with limited access), then rechecked after correction.
## Decommissioning
-  Avoid throwing out devices, either recycle them to another use, or **destroy them**.
-  if not managed formally and adequately, someone can access your data after you throw out the device.
# Hardening Techniques
## OS & System Hardening
- **Patching:** Apply monthly security updates to operating systems and test them before mass rollout.
- **Account Security:** Enforce **password policies** (complexity and length) and **least privilege**, ensuring users only have necessary permissions.
- **Application Hygiene:** **Delete unused software** to reduce the attack surface and simplify update management.
- **Allow/Deny Lists:** Restrict software execution based on **hashes, certificates, or paths**.
## Encryption & Data Protection
- **Storage Encryption:** Use **Encrypting File System (EFS)** for specific files or **Full Disk Encryption (FDE)** like BitLocker or FileVault for entire volumes.
- **In-Transit Protection:** Secure network traffic using **VPNs** or application-level encryption like **HTTPS**.
- **Decommissioning:** Properly **destroy** or recycle old hardware to prevent data leaks from discarded devices.
## Endpoint Detection & Monitoring
- **EDR:** Utilises **behavioral analysis** and machine learning to identify threats; it can autonomously isolate systems and **roll back** to previous configurations.
- **Host-Based Firewall:** Provides visibility of traffic **pre- and post-encryption** to block malicious processes.
- **HIPS:** Monitors for **buffer overflows**, registry changes, and unauthorised system file modifications.
- **SIEM:** Collects and manages logs from various sensors and firewalls for centralized monitoring.
## Network & Interface Security
- **Port Management:** Close unnecessary ports to minimize entry points; use **Nmap** to scan for open vulnerabilities.
- **Access Control:** Use **IP address ranges** or **ACLs** to restrict who can access specific servers.
- **Management Interfaces:** Always change **default passwords** and implement **multifactor authentication (MFA)** or centralised authentication.
- **Segmentation:** Improve security by separating the network physically or logically using **VLANs**.
