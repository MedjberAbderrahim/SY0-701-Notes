## Data at Rest
-  SSDs, HDDs, USB Drives, ... are considered **data at rest**
-  **Full Disk and Volume/Partition Encryption**: BitLocker, FileVault, ...
-  **File Encryption**: EFS (Encrypting File System) on NTFS, and other 3rd party tools...
## Database Encryption
-  **Transparent Encryption**: Encrypt **all** database information with a symmetric key.
-  Record-Level Encryption:
	-  Encrypt individual columns to reduce complexity.
	-  use separate symmetric keys for each column.
## Transport Encryption
-  Browsers generally use encrypted `HTTPs`
-  For secure remote access or connection between different sites, we use a VPN:
	-  Client based VPN using SSL/TLS.
	-  Site-to-Site VPN using IPsec.
## Key Lengths
-  Larger Keys tend to be more secure.
-  For symmetric Encryption, 128-bit keys or larger.
-  For asymmetric: Larger keys than symmetric, can go up to 3072-bits or more, depends on complex calculations of prime numbers.
## Key Stretching/Strengthening
Making a key secure by randomizing it even further (hashing it again and again, ...)
