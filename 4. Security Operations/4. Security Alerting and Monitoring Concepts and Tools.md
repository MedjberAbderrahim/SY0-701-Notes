## Monitoring Computing Resources
-  Monitoring is like Attackers, they never stop
-  Must always monitor & respond for all kinds of events, e.g. remote access, firewalls, ...
### Systems
-  Detect strange authentications, like logins from unrelated, foreign countries
-  Server Monitoring, i.e. services, backups, versions ...
### Applications
-  Must be monitored for their uptime, data transfer rates (higher intensity is usually suspicious) & security notifications
### Infrastructure
-  Monitor & Verify remote access systems
-  Check logs of firewalls, IPSs, ...; can be helpful in detecting anomalies
## Activities
### Log Aggregation
-  **SIEMs (Security Information & Event Management)** Consolidate & centralize different systems' logs into one source
-  Offers correlation between diverse systems & detailed analysis
### Scannning
-  Constant & continuous checking for new vulnerabilities in OSs, applications, ...
-  Large amounts of data generated, useful for later reports
### Reporting
-  Create "actionable" reports (reports that redirect us where to go further) from the output of scanning
-  Should contain extensive status information, so they can be referenced when new vulnerabilities are out.
-  **ad hoc reports:** Reports analyzing what-if scenarios, of possible future situations, example:
	Study case where we analyze systems' susceptibility to attacks once their OS hits EOL.
### Archiving
-  Data collection not only for business performance analysis, but also to detect attackers
### Alerting
-  Real-time alerting using actionable data is crucial for security operations
-  Alerts can be SMS, Emails, Security console notifications, ...
### Alert Response and Remediation Validation
#### Quarantine
-  Quarantine the system from the rest of the network to reduce lateral movement & propagation risks.
#### Alert Tuning
-  Alarming/Alerting should be as accurate as possible, to prevent false positives/negatives
-  We maximize that by adjusting alerts to the right pace of the network
## Tools
### SCAP (Security Content Automation Protocol)
-  Protocol made by NIST to standardize data of security tools (IPS, NGFW, ...)
-  Allows tools to generate & act on the same form of data, and collaborate together
- Very useful in large organizations, also allows for further automation
### Benchmarks
- Apply security best practices to all possible devices, considered the bare minimum for security settings
- Popular benchmarks: **CIS (Center for Internet Security)** - [cis-benchmarks](https://www.cisecurity.org/cis-benchmarks).
### Agents/agentless
- We can check a device's state of compliance using 2 methods:
	- **Agent (AV, EDR, ...)**: continuous, constant, and automatic check, agent must be installed/maintained on system
	- **Agentless**: on-demand, generally remote check, using existing protocols, doesn't require prior installation
### SIEM (Security Information & Event Management)
- Collects, aggregates, stores, correlates, and analyzes data/logs for better understanding of events
### Anti-Virus
- Solution against all kinds of malware
### DLP (Data Loss Prevention)
- Stop data leakage using a DLP solution, blocks certain types of data before going out the network
### SNMP (Simple Network Management Protocol)
- Polls devices for statistics at fixed intervals over UDP/161
- SNMP agents can send alerts (for urgent stuff) back to the management station on UDP/162, called SNMP Traps
