A threat vector is the method an attacker uses to access a system. As attackers innovate, security teams must identify both known and emerging threats.
### Exploitation of Communication Channels
Attackers target messaging systems like:
-  **Email:** Used for phishing, malware, or malicious links.
-  **SMS:** Lures users with fake alerts or links.
-  **Messaging Apps:** Platforms like WhatsApp or social media DMs for social engineering.

Phishing via these channels can lead to malware installation or data theft.
### Social Engineering & Scams

Attackers manipulate victims to reveal info or perform malicious actions, e.g., fake invoices or crypto scams. An example involved a USPS spam message with a malicious link.
### Image & Web Content Attacks

SVG images can embed malicious code, executing in browsers and causing XSS attacks. Browser protections help, but vulnerabilities exist.
### File-Based Threats

Malicious files include:
-  **Executables (.exe):** Run directly in memory.
-  **PDFs:** Can contain embedded scripts.
-  **Compressed Files (ZIP, RAR):** Obfuscate malware.
-  **Office Files:** Macros can be malicious.
-  **Browser Extensions:** Malicious add-ons compromise security.
### Mobile & Call System Threats

-  **Vishing:** Phone scams to steal info.
-  **Spam over IP:** Automated spam calls/messages.
-  **War Dialing:** Scanning for unprotected lines.

Messaging-based DoS attacks can also disrupt services.
### Physical & Hardware Threats

-  **USB Drives:** Used in air-gapped (No Direct Network Connection into the network) attacks or data theft.
-  **Modified Peripherals:** Devices that execute malicious commands.
### Software & Patch Risks

Outdated software has vulnerabilities. Regular updates reduce risks. Unsupported systems should be upgraded or decommissioned.
### Network & Access Risks

-  **Wireless Security:** Use WPA3; avoid WEP/WPA2.
-  **Rogue Access Points:** Unauthorized Wi-Fi points.
-  **802.1X:** Ensures device authentication.
-  **Open Ports:** Secure and patch services on common ports.
### Default Credentials & Supply Chain

Default passwords (e.g., admin/admin) pose risks. Change them. Supply chain threats include counterfeit hardware with vulnerabilities.
-  Default router passwords List:  **[Link](https://www.routerpasswords.com/)**
### Third-Party & MSP Risks

Vendor breaches, like Target’s HVAC system hack, show the importance of monitoring third-party access.
### Best Practices
1. Regularly patch systems.
2. Decommission unsupported devices.
3. Secure Wi-Fi with latest protocols and strong authentication.
4. Change default passwords.
5. Control access to open ports/services.
6. Monitor hardware integrity and supply chain.
7. Restrict physical access and USB connections.
8. Maintain an updated device inventory.

Understanding these vectors helps strengthen security defenses.
